★ Mobile First ★

Desenvolver primeiro para as plataformas móveis

A tradução literal do termo Mobile first é: móvel primeiro.

Este conceito foi apresentado em 2009 por Luke Wroblewski, que mais tarde escreveu o livro “Mobile First” sobre o mesmo tema. O livro é repleto de informações, e defende que os sites e aplicativos devem ser cada vez mais projetados para dispositivos móveis, descrevendo como as equipes de Web design podem fazer a transição do design para desktops / laptops ao design para dispositivos móveis. O livro também apresenta considerações de design para uma melhor organização das ações, entradas e layout da Web para dispositivos móveis.

Este paradigma emerge num contexto onde os dispositivos de comunicação móveis estão cada vez mais rápidos e mais utilizados pela da população mundial. Segundo Aleixo, mais de 1 bilhão de pessoas têm acesso à Internet e quase 3 bilhões de pessoas possuem um dispositivo móvel. Esses números são enormes e continuam a aumentar consideravelmente ano a ano, tornando a necessidade de desenvolvimento de aplicativos e sites mobile friendly um fator crucial no desenvolvimento dos negócios/empresas

Segundo Braga Lopes: “Os dados foram coletados pela Pesquisa Brasileira de Mídia (PMB) 2015. De acordo com o levantamento, mais de metade da população brasileira tem acesso à internet, com aumento dário graças a difusão de smartphones e tablets no país. Onde, atualmente 66% dos acessos feitos a internet são realizados via smartphones, contra 40% no ano anterior. Além disso, o número de acessos diários aumentou para 76% em 2014, contra 56% em 2013”.

Os projetos que utilizam este novo paradigma fazem a criação de produtos web, sites e aplicativos pensando primeiro na arquitetura e interfaces para os dispositivos móveis e então fazem adaptações para o desktop e outras plataformas.

tableless 2015

Na imagem anterior podemos notar a diferença do enfoque tradicional comparado ao Mobile First.

Pensando num enfoque tradicional a arquitetura e funcionalidades pensadas para dispositivos desktop são bastante livres e requerem pouca sintetização de conteúdo, pois as telas de dispositivos desktop são maiores e tem-se mais liberdade com relação ao posicionamento dos componentes;

Quando trata-se de adequar esse mesmo conteúdo para as plataformas móveis os designers encontram o desafio: o que é o principal, quais são as funcionalidades mais utilizadas e como devo organizar-las numa tela menor?

O enfoque mobile first, desktop second facilita essa transição, pois quando projetamos primeiro para telas menores somos forçados a sermos sucintos e fazer melhor uso da interface para oferecer a melhor experiência de usuário. Feito isso a transição para desktop é mais orgânica, e costuma trazer benefícios gráficos inclusive.

Os dispositivos móveis possuem uma tela menor, mas uma infinidade de outras facilidades que podem ser utilizadas pelos desenvolvedores tais como: acelerômetros, cameras, gravadores de áudio, sensores e etc. Esses dispositivos impulsionam o desenvolvimento de funcionalidades inovadoras e amplificam a interatividade dos sites e aplicativos, proporcionando um grande avanço conceitual na área de design e UX.

As imagens a anteriores demonstram como a visão mobile first pode influenciar não apenas aplicações mobile mais confortáveis e com boa experiência de usuário mas como essas qualidades se estendem aos aplicativos e sites desktops; uma vez que o conteúdo precisa ser pensado e avaliado, tanto em frequência de utilização, pertinência, fluxo de passos a serem efetuados para uma jornada completa no aplicativo/ site

Ao planejar um site ou aplicativo mobile/desktop, os desenvolvedores, arquitetos e designers precisam atentar-se às diferenças de interação: linguagem natural, linguagens de comando, menus, WIMP (windows, icons, menus and pointers) e etc. estas diferenças são muito importante nas adequações que se fazem nas interfaces.

Segundo Jeffrey Zeldman “Every device does not need to have the same experience. Trying to maintain the same experience in all devices is dated. It is an obsolete approach. People understand different devices provide different experiences.”

Zeldman, enfatiza que diferentes dispositivos não precisam nem devem conter as mesmas informações e mesmas experiências, cada interação precisa ser pensada no contexto do dispositivo. Muitas dicas de como fazer podem ser aplicados quando se trata de Mobile First e sites responsivos; Citando algumas dicas para uma boa UX, podemos elencar a responsividade em si como essencial.

Mesmo pensando no mobile primeiro, é interessante que o mesmo site seja adaptado para diferentes dispositivos, ao invés de criar várias versões. Assim não há links diferentes, e o próprio site se adapta ao dispositivo do usuário, apenas o conteúdo varia em cada resolução.

Pensando num site responsivo para mobile primeiro e depois desktop, faz todo sentido que o carregamento também deve ser progressivo, sendo assim ao invés de ocultarmos conteúdo para os dispositivos com telas menores, nós devemos fazer o contrário e carregar inicialmente apenas o conteúdo para os menores dispositivos e então progressivamente carregar os conteúdos para as maiores telas. poupando o maior bem dos usuários de dispositivos móveis que é o uso de dados

Todas essas dicas podem ser encontradas com facilidade na internet, entretanto o maior desafio é de fato a mudança de paradigma, pois atualmente, a maioria das empresas ainda desenvolve os seus sites pensando no desktop como tela principal. Mas isso provavelmente é uma questão de tempo, pouco tempo até isso mudar
